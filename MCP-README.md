# Accessibility MCP Server\n\n## Overview\n\nThe Accessibility MCP (Model Context Protocol) Server provides real-time accessibility guidelines by scraping current information from authoritative sources instead of relying on static S3 JSON files. This ensures that your Bedrock AI agent always has access to the most up-to-date accessibility guidance.\n\n## Architecture\n\n```\n┌─────────────────┐    ┌─────────────────┐ when design systems change\n\n### 🎯 **Comprehensive Coverage** \n- GOV.UK Design System guidelines\n- W3C WCAG 2.2 compliance requirements\n- ARIA specification details\n- Covers ALL accessibility rules, not just a subset\n\n### 💾 **Intelligent Caching**\n- 24-hour TTL for performance\n- Fallback to S3 when scraping fails\n- Background cache maintenance\n\n### 🔄 **Seamless Integration**\n- Drop-in replacement for existing Bedrock setup\n- Backwards compatible with S3 fallback\n- Zero breaking changes to current workflow\n\n## Components\n\n### 1. MCP Server Core\n```\nsrc/test/java/org/dvsa/testing/framework/mcp/\n├── AccessibilityMcpServer.java          # Main MCP server\n├── models/\n│   └── AccessibilityGuideline.java       # Data model for guidelines\n├── scrapers/\n│   ├── DesignSystemScraper.java          # GOV.UK scraper\n│   └── WcagScraper.java                  # W3C WCAG/ARIA scraper\n├── cache/\n│   └── GuidelineCache.java               # TTL-based caching\n├── client/\n│   └── AccessibilityMcpClient.java       # Client for integration\n├── config/\n│   └── McpConfig.java                    # Configuration\n└── integration/\n    └── McpEnhancedBedrockAnalyser.java   # Enhanced Bedrock integration\n```\n\n### 2. Available MCP Tools\n\n**`scrape_design_system_guidelines`**\n- Parameters: `component_type` (forms, tables, buttons, etc.)\n- Returns: Current GOV.UK Design System accessibility guidance\n\n**`get_aria_guidance`**  \n- Parameters: `aria_attribute`, `element_role`\n- Returns: W3C ARIA specification details for specific attributes\n\n**`check_wcag_compliance`**\n- Parameters: `rule_id`, `level` (A, AA, AAA)\n- Returns: WCAG 2.2 compliance requirements and techniques\n\n**`refresh_guidelines_cache`**\n- Parameters: `force_refresh` (boolean)\n- Returns: Cache refresh confirmation\n\n## Setup & Installation\n\n### 1. Start the MCP Server\n\n```bash\n# Make script executable (if not already)\nchmod +x scripts/start-mcp-server.sh\n\n# Start the server\n./scripts/start-mcp-server.sh\n```\n\nThe server will start on `localhost:8080` by default and begin scraping real-time guidelines.\n\n### 2. Configure Your Application\n\nAdd these properties to your `application.conf`:\n\n```properties\n# Enable MCP for real-time guidelines\nbedrock.agent.knowledge.source=mcp\nbedrock.agent.fallback.to.s3=true\n\n# MCP Server settings\nmcp.server.host=localhost\nmcp.server.port=8080\nmcp.cache.enabled=true\nmcp.cache.ttl.hours=24\n```\n\n### 3. Update Your Code\n\nReplace your existing `BedrockAgentAnalyser` with the MCP-enhanced version:\n\n```java\n// OLD: Static S3 knowledge base\n// BedrockAgentAnalyser analyser = new BedrockAgentAnalyser(client);\n\n// NEW: Real-time MCP guidelines\nMcpEnhancedBedrockAnalyser analyser = new McpEnhancedBedrockAnalyser(client);\n\n// Same API, enhanced functionality\nList<BedrockRecommendation> recommendations = \n    analyser.analyseViolationsWithBedrock(violations, emptyKbMap);\n```\n\n## Example: Before vs After\n\n### Before (Static
